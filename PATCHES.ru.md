# Mattermost Team Edition — Кастомные патчи

Документ описывает все изменения, внесённые поверх официального релиза
Mattermost Team Edition (`v11.8.1`). Патченый образ публикуется как
`mattermost:latest` / `mattermost:v11.8.1-patched`.

---

## 1. Тёмная тема — синхронизация с ОС

**Файлы:** `webapp/channels/src/components/theme_provider/`

Добавляет автоматическое переключение светлой/тёмной темы вслед за системными
настройками (macOS / Windows / Linux).

- При переключении ОС в тёмный режим интерфейс переходит на тему **Onyx**.
- При возврате в светлый — восстанавливается сохранённая пользователем светлая
  тема (по умолчанию Quartz).
- В Настольном приложении (Electron) используется нативный API `onDarkModeChanged`
  и `getDarkMode()`. Для версий Electron, где нативное событие ненадёжно,
  добавлен фолбэк через `matchMedia` и опрос каждые 2 секунды.
- В браузере используется `window.matchMedia('(prefers-color-scheme: dark)')`.
- Тема применяется **на уровне модуля** (до рендера React) через снимок
  `localStorage`, чтобы не было мигания неправильной темой при жёстком
  обновлении страницы.
- Системная консоль всегда отображается в светлой теме независимо от ОС.
- В **Настройки → Вид → Тема** добавлен чекбокс «Синхронизировать с темой ОС».

**По умолчанию для новых пользователей:** синхронизация **включена**.  
При создании аккаунта сервер сохраняет
`display_settings / sync_with_os_theme = "true"`.

## 3. Встроенная интеграция LDAP / Keycloak

**Файлы:** `server/channels/app/ldap.go`, `server/channels/api4/ldap.go`,
`server/channels/app/keycloak_ldap.go`, `server/channels/app/channels.go`,
`server/channels/app/server.go`

Заменяет корпоративный LDAP-плагин встроенной реализацией, работающей в
Team Edition без лицензии.

### 3а. Вход через LDAP

- Стандартная LDAP-аутентификация (bind + поиск) без проверки лицензии.
- `TestLdapConnection` дополняет поле `BindPassword` из сохранённой конфигурации
  сервера, если системная консоль отправляет пустое значение (исключает ложные
  ошибки при тесте соединения).
- `SyncLdap` запускает задачу синхронизации через встроенный планировщик задач
  при отсутствии корпоративного плагина.

### 3б. Синхронизация групп Keycloak

- Получает группы и их участников через **Keycloak Admin REST API**.
- При старте сервера группы Keycloak синхронизируются с группами Mattermost
  и распространяются на команды и каналы.
- Вспомогательная функция `ldapGroupsAllowed()`: возвращает `true`, если
  провайдер является Keycloak (без проверки лицензии) или если есть лицензия с
  поддержкой `LDAPGroups`.
- Проверки лицензии удалены из обработчиков API: `syncLdap`, `testLdap`,
  `testLdapConnection`, `testLdapDiagnostics`, `migrateIDLdap`.

## 4. Провайдер OpenID Connect (JWKS / JWT)

**Файлы:** `server/channels/app/oauthproviders/openid/openid.go`

Расширяет встроенный провайдер OpenID Connect:

- **Кеширование JWKS** — набор публичных ключей загружается один раз и
  кешируется, чтобы не запрашивать провайдер при каждой верификации токена.
- **Разбор и верификация JWT** — проверка подписи RS256 / ES256 по кешированным
  ключам JWKS.
- **Дополнение данных** — если эндпоинт `userinfo` возвращает неполные данные,
  недостающие поля берутся из JWT-клеймов (полезно, когда Keycloak не
  публикует все скоупы на `userinfo`).
- Проверка клейма `sub` для корректной привязки пользователя.

**Файлы:** `server/public/model/license.go`,
`server/channels/app/platform/license.go`

Включает функции, заблокированные лицензией Enterprise в стоковой Team Edition,

## 6. Умолчания для новых пользователей

При создании аккаунта автоматически сохраняются следующие настройки
(`server/channels/app/user.go` → `createUserOrGuest`):

| Категория | Имя | Значение | Эффект |
|-----------|-----|---------|--------|
| `display_settings` | `sync_with_os_theme` | `true` | Синхронизация с темой ОС включена (§1) |
| `display_settings` | `click_to_reply` | `false` | Клик по сообщению **не** открывает тред автоматически |

### Открытие треда только по кнопке Reply

В стоковом Mattermost клик по любому месту сообщения открывает тред в боковой
панели. Патч устанавливает `click_to_reply = false`, чтобы панель открывалась
только по явному нажатию **Reply**. Пользователь может изменить это в
**Настройки → Вид → Открывать треды по клику**.

## 7. Боковая панель треда — увеличенный лимит ширины

**Файл:** `webapp/channels/src/sass/layout/_sidebar-right.scss`

В стоковом приложении боковая панель треда/ответов ограничена по ширине:
400–776 пикс. в зависимости от размера экрана. Патч заменяет все фиксированные
значения `max-width` на `50vw`, позволяя растянуть панель до **половины ширины
экрана** на любом разрешении.

Минимальная ширина (304 пикс. на широких экранах) сохранена для удобства.

## 8. Информация о сборке в диалоге «О Mattermost»

**Файлы:** `Dockerfile.team-patch`, `gcp/cloudbuild.yaml`

Стандартная Team Edition оставляет поля **Номер сборки**, **Хеш сборки**
и **Дата сборки** пустыми. Патч вшивает их в бинарник через ldflags Go при
компиляции:

| Поле | Источник |
|------|---------|
| Build Number | Количество коммитов в ветке (`git rev-list --count HEAD`) |
| Build Hash | Полный SHA коммита — 40 символов (однозначно идентифицирует образ в Artifact Registry) |
| Build Date | UTC-метка времени сборки (`YYYY-MM-DDThh:mm:ssZ`) |

Применяется к бинарникам `mattermost` и `mmctl`.

## 9. Docker-сборка и CI/CD

**Файлы:** `Dockerfile.team-patch`, `gcp/cloudbuild.yaml`

### Dockerfile

Многоэтапная сборка (`Dockerfile.team-patch`):

| Этап | Базовый образ | Результат |
|------|--------------|----------|
| `webapp-builder` | `node:24-bookworm-slim` | Скомпилированные JS/CSS ресурсы |
| `server-builder` | `golang:1.26.4-alpine` | Бинарники `mattermost` и `mmctl` |
| `runtime` | `mattermost/mattermost-team-edition:11.8` | Финальный образ |

Финальный этап заменяет только серверные бинарники и ресурсы веб-приложения;
всё остальное (плагины, i18n, шаблоны конфигурации) берётся из официального
образа.

Ассерты в `server-builder` явно прерывают сборку, если любой из бинарников не
скомпилирован с Go 1.26.4.

### Cloud Build

Триггер: пуш тегов по маске `v*-patched` (прод) и `v*-patched-dev` (дев).

- Прод-теги → образ с тегами `:v11.8.1-patched` + `:latest`
- Дев-теги → образ с тегом `:dev` + дополнительный тег

Кеш BuildKit registry (`buildcache`) используется для этапа `webapp-builder`
(npm build ~8-12 мин). Этапы `server-builder` и `runtime` всегда пересобираются
с нуля (`--no-cache-filter=server-builder,runtime`), чтобы кеш не подсунул
устаревший бинарник Go.

Уведомления о начале/успехе/ошибке сборки отправляются в канал Mattermost через
API (токен-авторизация) со ссылками на лог Cloud Build и образ в Artifact
Registry.

## 10. Безопасность — исправление CVE в стандартной библиотеке Go

**Файлы:** `Dockerfile.team-patch`, `server/go.mod`, `server/public/go.mod`,
`server/.go-version`

Тулчейн зафиксирован на **Go 1.26.4** (вышел 2026-06-02), который содержит
исправления стандартной библиотеки:

| CVE | Серьёзность | Пакет | Описание |
|-----|------------|-------|---------|
| CVE-2026-42504 | High | `mime` | Квадратичная нагрузка CPU в `WordDecoder.DecodeHeader` |
| CVE-2026-27145 | Medium | `crypto/x509` | Квадратичный `VerifyHostname` при большом числе SAN |
| CVE-2026-42507 | Medium | `net/textproto` | Неэкранированный ввод в сообщениях об ошибках |

Оба бинарника (`mattermost` и `mmctl`) компилируются с этим тулчейном, поэтому
ни один не содержит уязвимой stdlib.

Оставшиеся находки (glibc, openssl) приходят из базового образа
`mattermost-team-edition:11.8` и не имеют апстрим-фикса.
